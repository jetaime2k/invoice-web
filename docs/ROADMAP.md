# 노션 견적서 뷰어 고도화 로드맵

노션 견적서 뷰어 MVP 완료 후, 관리자 경험과 클라이언트 공유 플로우를 개선하는 Phase 5 고도화 로드맵

## 개요

MVP(Phase 1–4, Task 001–011) 완료를 기반으로, 발행자의 일상적인 사용 경험을 한 단계 높이는 세 가지 고도화 작업을 수행합니다:

- **다크 모드 완전 구현**: next-themes로 시스템 테마와 연동되는 라이트/다크 전환
- **관리자 레이아웃 개선**: 사이드바 네비게이션, 통계 카드로 대시보드 경험 강화
- **견적서 목록에서 공유 링크 복사**: 목록 페이지에서 바로 클라이언트 링크 발급 및 복사

## 개발 워크플로우

1. **작업 계획**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입

2. **작업 생성**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - `/tasks` 디렉토리에 새 작업 파일 생성
   - 명명 형식: `XXX-description.md` (예: `012-dark-mode.md`)
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - **API/비즈니스 로직 작업 시 "## 테스트 체크리스트" 섹션 필수 포함 (Playwright MCP 테스트 시나리오 작성)**
   - 예시를 위해 `/tasks` 디렉토리의 마지막 완료된 작업 참조

3. **작업 구현**
   - 작업 파일의 명세서를 따름
   - 기능과 기능성 구현
   - **API 연동 및 비즈니스 로직 구현 시 Playwright MCP로 테스트 수행 필수**
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트
   - 구현 완료 후 Playwright MCP를 사용한 E2E 테스트 실행
   - 테스트 통과 확인 후 다음 단계로 진행
   - 각 단계 완료 후 중단하고 추가 지시를 기다림

4. **로드맵 업데이트**
   - 로드맵에서 완료된 작업을 ✅로 표시

  - ✅ 대시보드 헤더 컴포넌트 `src/components/dashboard/header.tsx` — 페이지 제목, ThemeToggle, 사용자 이메일 표시
  - ✅ 통계 카드 컴포넌트 `src/components/dashboard/stats-cards.tsx` — 총 견적서 수, 이번 달 발행 건수, 마지막 동기화 시간
  - ✅ `(dashboard)/layout.tsx` 재구성 — 사이드바 + 메인 영역 2-컬럼 레이아웃 적용
  - ✅ 견적서 목록 페이지 상단에 통계 카드 3개 삽입
  - ✅ 모바일 반응형 — 사이드바 콜랩스 처리 (오버레이 드로어)
  - **수락 기준**:
    - 사이드바에서 견적서 목록·설정 페이지 이동 작동
    - 현재 활성 경로가 사이드바에서 시각적으로 구분됨 (`usePathname` 활용)
    - 통계 카드에 총 견적서 수, 마지막 동기화 시간 실제 데이터 표시
    - 375px 모바일 뷰포트에서 사이드바 숨김 처리 후 접근 가능
    - 다크 모드에서 사이드바·카드 배경색 정상 적용 (Task 013 연계)

- **Task 013: 다크 모드 완전 구현** ✅ - 완료
  - ✅ ThemeProvider 래핑 확인 — `attribute="class"`, `defaultTheme="system"`, `enableSystem` 설정 검증 (`src/app/layout.tsx`)
  - ✅ `globals.css` `.dark` 블록 완성 — oklch 다크 팔레트 전체 토큰 점검 및 누락 변수 보완
  - ✅ `(dashboard)/layout.tsx` 헤더에 `<ThemeToggle />` 배치
  - ✅ `(auth)/layout.tsx` 로그인·회원가입 페이지 다크 배경 대응
  - ✅ 공유 페이지 `/share/[token]` 다크 모드 색상 확인 (amber 경고 박스 포함)
  - ✅ 설정 페이지 `/settings` 다크 모드 적용
  - **수락 기준**:
    - ThemeToggle 클릭 시 라이트 → 다크 → 시스템 전환 작동
    - 새로고침 후에도 선택한 테마가 유지됨 (localStorage 기반)
    - 시스템 다크 모드 변경 시 자동 반영
    - 모든 페이지에서 하드코딩 배경색 없음 — CSS 변수 전용 사용
    - 공유 페이지 amber 경고 박스 다크 색상 정상 적용

- **Task 014: 견적서 목록에서 공유 링크 복사** ✅ - 완료
  - ✅ `getQuotes()` 수정 — 각 Quote에 활성 공유 링크 토큰을 LEFT JOIN하여 반환 (`share_links` WHERE `is_active = true`)
  - ✅ 클라이언트 컴포넌트 `src/components/quotes/copy-link-button.tsx` 구현 — `quoteId`, 초기 `shareToken`을 props로 받음
  - ✅ 토큰 없을 때: `createShareLink(quoteId)` 호출 후 생성된 토큰을 state에 저장 → 클립보드 복사
  - ✅ 토큰 있을 때: `navigator.clipboard.writeText()` 즉시 호출
  - ✅ sonner toast로 성공("링크 복사됨") / 실패("복사 실패 — 브라우저 권한을 확인하세요") 피드백
  - ✅ 견적서 목록 테이블에 "링크" 컬럼 추가 — `CopyLinkButton` 렌더링
  - **수락 기준**:
    - 목록 각 행에 복사 버튼이 표시됨
    - 공유 링크 없는 견적서: 버튼 클릭 시 자동 생성 후 "링크 복사됨" 토스트 표시
    - 공유 링크 있는 견적서: 즉시 클립보드에 복사 후 토스트 표시
    - 복사된 URL로 `/share/[token]` 접근 시 견적서 정상 표시
    - 기존 상세 페이지 `ShareLinkBox` 동작에 영향 없음 (회귀 없음)
