아래 "메타 프롬프트 본문"을 그대로 복사하여 `prd-generator` 에이전트에게 전달하세요.

```
서브에이전트(`.claude/agents/docs/prd-generator.md`)를 사용해서 아래 프로젝트의 1인 개발자용 MVP PRD를 만들어줘.
```

## 메타 프롬프트 본문

> 노션(Notion) 데이터베이스에 입력한 견적서 내용을 클라이언트가 웹에서 조회하고, PDF로 다운로드할 수 있는 서비스를 위한 1인 개발자용 PRD를 만들어줘.
>
> ### 프로젝트 핵심
>
> - **목적**: 견적서 발행자가 노션에 작성한 견적서를 클라이언트가 별도 프로그램 없이 웹에서 바로 확인하고, 필요시 PDF로 저장할 수 있게 한다.
> - **타겟 사용자**:
>   - **발행자(관리자)**: 노션 데이터베이스에 견적서를 작성/관리하는 프리랜서 또는 소규모 사업자
>   - **클라이언트(열람자)**: 발행자로부터 받은 링크로 견적서를 조회하는 고객
>
> ### 데이터 연동 방식
>
> - 견적서 원본 데이터는 **노션 데이터베이스**에서 작성 및 관리한다.
> - 웹 서비스는 **Notion API**를 통해 견적서 데이터를 읽기 전용으로 조회하여 화면에 표시한다.
> - 발행자(관리자)는 로그인 후 자신이 등록한 견적서 목록을 확인할 수 있다.
> - 클라이언트(열람자)는 별도 회원가입/로그인 없이, 발행자가 공유한 **고유 링크**를 통해 해당 견적서를 조회한다.
>
> ### PDF 다운로드 방식
>
> - 서버 사이드 렌더링 없이, **브라우저에서 클라이언트 사이드 라이브러리**(예: react-pdf, html2canvas + jsPDF 등)를 사용해 견적서 화면을 PDF로 변환하여 다운로드한다.
>
> ### 기술 스택 컨텍스트
>
> - 본 프로젝트는 Next.js 15.5.3 + React 19 + TypeScript + TailwindCSS v4 + shadcn/ui 기반으로 개발한다.
> - PRD의 기술 스택 섹션도 이 스택을 기준으로 작성하되, Notion API 연동 및 클라이언트 사이드 PDF 변환에 필요한 라이브러리를 포함해줘.

## 참고 사항

- `prd-generator`는 개발 우선순위, API 라우트, 인프라, 마일스톤, 보안 요구사항 등은 생성하지 않습니다. 이는 의도된 동작입니다.
- 생성된 PRD는 `docs/PRD.md`로 저장하세요. (`CLAUDE.md`의 `@/docs/PRD.md` 참조와 연결됩니다.)
- 이후 단계:
  1. `prd-validator` 에이전트로 PRD의 기술적 타당성을 검증
  2. `development-planner` 에이전트로 `docs/ROADMAP.md` 생성
